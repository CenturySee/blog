强烈建议正在研发Agent系统的工程师看一下Anthropic的engineer系列博客，博客内容是真的在解决生产环境的实际问题。

> https://www.anthropic.com/engineering

几乎每一篇都是必读之作。  

这篇文章仅对列表里最具代表性的文章做一个简要介绍，具体内容大家可根据链接访问原文。

## [Contextual Retrieval in AI Systems](https://www.anthropic.com/engineering/contextual-retrieval)

介绍了一种名为“上下文检索”（Contextual Retrieval）的方法，该方法通过在生成 Embeddings 和创建 BM25 索引之前将特定于块的上下文添加到每个块中（Contextual Embeddings 和 Contextual BM25），解决了传统 RAG 系统因切分文档导致上下文丢失从而检索失败的问题。

## [Building Effective AI Agents](https://www.anthropic.com/engineering/building-effective-agents)

总结了构建有效 AI Agent 的模式，提出了从增强型 LLM 到复杂工作流（如 Prompt Chaining、Routing、Parallelization、Orchestrator-workers、Evaluator-optimizer）再到自主 Agent 的逐步构建方法，解决了在不同场景下如何平衡系统复杂性、可控性与灵活性的问题。

## [The "think" tool: Enabling Claude to stop and think](https://www.anthropic.com/engineering/claude-think-tool)

介绍了一种“思考”工具（think tool），允许 Claude 在采取行动前输出思考过程，通过赋予模型结构化思考的空间，解决了复杂任务中（如遵循复杂策略或多步推理）模型决策不一致和错误率高的问题。

## [Claude Code Best Practices](https://www.anthropic.com/engineering/claude-code-best-practices)

提供了一系列使用 Claude Code 的最佳实践，通过定制化环境配置、优化工具使用、采用特定工作流（如 TDD、视觉反馈迭代）以及利用多 Agent 协作等方法，解决了开发者在使用 Agentic coding 工具时面临的学习曲线和效率问题。

## [How we built our multi-agent research system](https://www.anthropic.com/engineering/multi-agent-research-system)

介绍了一种多 Agent 研究系统的架构，通过采用 Orchestrator-Worker 模式，由主 Agent 协调多个子 Agent 并行搜索和综合信息，解决了单 Agent 在处理需要广泛搜索和大量上下文的复杂研究任务时效率低下和上下文受限的问题。

## [Writing effective tools for AI agents—using AI agents](https://www.anthropic.com/engineering/writing-tools-for-agents)

探讨了如何编写高效的 Agent 工具，通过利用 Agent 自身来构建原型、生成评估任务、分析结果并优化工具定义，解决了传统软件开发模式不适应非确定性 Agent 交互的问题，提高了 Agent 调用工具的准确性和效率。

## [Building agents with the Claude Agent SDK](https://www.anthropic.com/engineering/building-agents-with-the-claude-agent-sdk)

介绍了 Claude Agent SDK，通过为 Agent 提供计算机访问权限（文件系统、终端）以及支持 Agentic search、Subagents 和 Compaction 等功能，解决了构建能够执行复杂、多步骤工作流的通用 Agent 的工程难题。

## [Effective context engineering for AI agents](https://www.anthropic.com/engineering/effective-context-engineering-for-ai-agents)

提出了“上下文工程”（Context Engineering）的概念，通过系统提示（System Prompts）、工具优化、示例精选以及动态上下文检索（如 Just-in-time retrieval、Compaction、Structured note-taking）等策略，解决了 LLM 有限注意力预算与海量信息之间的矛盾，确保 Agent 在长窗口任务中保持高效与连贯。

## [Equipping agents for the real world with Agent Skills](https://www.anthropic.com/engineering/equipping-agents-for-the-real-world-with-agent-skills)

介绍了“Agent Skills”机制，通过以文件夹形式组织指令（SKILL.md）和代码，利用渐进式披露（Progressive Disclosure）原则，解决了通用 Agent 缺乏领域特定知识且难以在有限上下文中加载大量专业能力的问题。

## [Making Claude Code more secure and autonomous with sandboxing](https://www.anthropic.com/engineering/claude-code-sandboxing)

介绍了 Claude Code 的沙盒机制（Sandboxing），通过文件系统隔离和网络隔离技术，在保障安全的前提下减少了频繁的权限确认请求，解决了 Agent 在自主执行任务时的安全性与用户体验（权限疲劳）之间的平衡问题。

## [Code execution with MCP: building more efficient AI agents](https://www.anthropic.com/engineering/code-execution-with-mcp)
提出了一种通过代码执行与 MCP 结合的方法，让 Agent 编写代码来调用 MCP 工具并处理数据，而不是直接通过 LLM 进行工具调用和上下文传输，解决了大规模使用 MCP 工具时造成的上下文窗口膨胀和中间结果冗余的问题。

## [Introducing advanced tool use on the Claude Developer Platform](https://www.anthropic.com/engineering/advanced-tool-use)

介绍了 Claude 开发者平台上的三项新功能：Tool Search Tool（按需发现工具）、Programmatic Tool Calling（通过代码编排工具调用）和 Tool Use Examples（提供工具使用示例），解决了大规模工具库中的上下文消耗大、执行效率低以及参数使用不准确的问题。

## [Effective harnesses for long-running agents](https://www.anthropic.com/engineering/effective-harnesses-for-long-running-agents)

介绍了一种用于长时运行 Agent 的 Harness 设计，通过分离初始化 Agent（Initializer Agent）和编码 Agent（Coding Agent），并利用持久化进度文件（claude-progress.txt）和 Git 历史来传递状态，解决了 Agent 跨多个上下文窗口工作时记忆丢失和任务连贯性中断的问题。
