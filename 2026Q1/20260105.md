# Google Antigravity：模型无法选择？补全这个配置就行

大家好，我是算法工程笔记。

前两天刚跟大家分享了[如何顺利用上 Google Antigravity](https://mp.weixin.qq.com/s/NettDwhw6R8JW2SCBoRpxg)，本以为配置好了就能愉快地 Coding，没想到第二天就翻车了——打开 IDE 发现**无法选择模型**。

折腾了一番，终于在一个技术社区找到了解法。今天这篇主要聊聊如何解决这个问题，希望能帮大家避个坑。

---

# 问题的根源

简单来说，Antigravity 在运行过程中依赖一个语言服务进程（Language Server）来处理代码理解和补全。如果你使用了 **Proxifier** 这类工具代理网络流量，但没有将这个语言服务进程加入代理规则，就会导致 IDE 无法连接到模型服务。

表现就是：IDE 里右侧的 Model 下拉菜单是空的，或者一直转圈圈。

# 解决方案

解决方法非常简单，只需要在你的代理软件（如 Proxifier）规则中，把遗漏的进程加上即可。

## Windows 用户

在 Proxifier 的 Rules 中，找到你为 Antigravity 设置的规则，在 Applications 列表里补充添加：

`language_server_windows_x64.exe`

## Mac 用户

如果你是在 macOS 上运行，对应的进程名称会有所不同，请添加：

`language_server_macos_arm` 

或者（取决于你的芯片架构）：

`language_server_macos_x64`

> **一句话总结**：把 `language_server` 相关的可执行文件加入到你的代理白名单里，让它能顺畅联网。
