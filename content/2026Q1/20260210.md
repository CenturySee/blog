# 躺在床上写代码？极致“去 IDE 化”开发流，手机+AI 搞定全栈发布

**摘要**：告别沉重的 IDE，也不用打开电脑。知名开发者 yetone 利用最近爆火的 "OpenClaw"，在 Telegram 上通过纯对话完成了两个真实项目的迭代与发布。这究竟是噱头，还是编程的未来？

---

就在昨天，`avante.nvim` 的作者 **yetone** 分享了他最新的开发体验：他利用 **OpenClaw**，在休假期间、完全不接触电脑、仅使用手机版 Telegram 的情况下，完成了两个软件项目（Alma 和 Confirmo）的功能迭代、测试。

他还说到：“去年实现了不写代码（靠 AI 生成），今年要实现不人工验证（靠 AI 跑测试、发截图）。”

这听起来像科幻小说，但其实是一套逻辑严密的 **ChatOps** 工程实践。本文将为你拆解这套“手机驱动全栈开发”背后的具体实现逻辑。

### 01 核心架构：当 Telegram 变成 IDE

这套系统的核心理念是：**将“指令”与“执行”彻底分离。**

* **前端（指挥官）**：**Telegram**。
作者利用 Telegram 的 **Topic（话题）** 功能实现了多任务并行。每一个 Topic 就是一个独立的“项目空间”。
* *Topic A*: 负责 macOS 桌面应用 Alma 的开发。
* *Topic B*: 负责 Web 服务 Confirmo 的后端逻辑。
这样做的好处是上下文（Context）天然隔离，互不干扰。


* **后端（执行者）**：**OpenClaw**。
运行在云端或本地高性能服务器上的 AI Agent。它不仅仅是一个聊天机器人（Chatbot），更是一个拥有“手脚”的超级工程师。它被授予了以下权限：
* **文件系统读写**：直接修改代码库。
* **Shell 执行**：运行编译命令、Git 操作、部署脚本。
* **浏览器/模拟器控制**：用于打开网页或 App 进行视觉验证。


### 02 工作流解析：90+% 的自动化是如何实现的？

传统的 AI 辅助编程（如 Copilot）流程是：
`你写需求 -> AI 生成代码 -> 你复制到 IDE -> 你运行 -> 报错 -> 你把错误贴回 AI`。
这种“人肉 ⌘C/⌘V”的过程极其痛苦，且无法脱离电脑。

Yetone 的 OpenClaw 流程则是完全的闭环：

#### 第一步：需求下达 (Coding)

作者在 Telegram 中输入：“给 Confirmo 增加一个 Telegram 通知功能，当任务完成时推送到我的手机。”

#### 第二步：自主编码与逻辑验证 (Logic Verification)

OpenClaw 接收指令后，自动分析代码库，修改后端逻辑。**关键点来了：**
Agent 修改完代码后，会**自动编写并运行单元测试**。

* 如果测试挂了？Agent 读取错误日志 -> 分析原因 -> 修正代码 -> 重新运行。
* 这个过程在服务器端自动循环，直到测试全部通过（Pass）。
* 作者不需要介入任何中间报错环节。

#### 第三步：视觉验收 (UI Verification)

对于 Alma 这种 macOS 桌面应用，或者 Web 页面，光有代码通过测试还不够，界面可能丑得没法看。
OpenClaw 的解决方案极其暴力且有效：**截图**。

1. Agent 启动编译好的 Alma 应用或打开 Confirmo 网页。
2. Agent 调用系统截图工具（或 Playwright），截取当前运行画面。
3. **Agent 把截图发回 Telegram Topic。**

作者拿着手机，点开大图：

* “按钮颜色太浅了，改深一点。”
* “布局歪了，向左移 10px。”

Agent 收到反馈，继续修改，再发新截图，直到作者回复：“通过（LGTM）”。

#### 第四步：一键发布 (Deployment)

验收通过后，OpenClaw 直接执行 Git Commit，打 Tag，触发 CI/CD 流水线，应用自动发布上线。

### 03 案例中的两个主角：Alma 与 Confirmo

为了证明这套流程不是“玩具”，作者展示了两个截然不同的项目：

* **Alma (客户端开发代表)**
这是一款 macOS 原生桌面应用（推测为 AI 辅助或知识库工具）。OpenClaw 展示了它不仅能写代码，还能搞定复杂的本地编译环境（Swift/Rust 等）和桌面 UI 的调整。
* **Confirmo (服务端开发代表)**
这是一款 Web 服务/工具。在演示中，OpenClaw 成功为其增加了 Webhook 功能，实现了业务逻辑与 Telegram 的打通。这证明了 Agent 在处理后端 API、数据库和第三方集成时的能力。

### 04 为什么这很重要？

这不仅是“懒人”的胜利，更是 **Agentic Workflow（代理工作流）** 的一次教科书级示范。

1. **从 Copilot 到 Autopilot**：
我们正在从“与 AI 结对编程”向“管理 AI 员工”转变。人类的角色从 **Writer（编写者）** 变成了 **Reviewer（审核者）** 和 **Decision Maker（决策者）**。
2. **屏幕解放**：
只要能收发消息，就能控制复杂的开发流程。这打破了“开发必须坐班”的物理限制。
3. **测试驱动开发 (TDD) 的复兴**：
为了让 AI 能自主验证，必须拥有完善的测试用例。这反而倒逼开发者去完善项目的自动化测试基建。

### 结语

Yetone 用 OpenClaw 证明了：当 AI 拥有了执行环境（Terminal）和视觉反馈（Vision）能力后，IDE 不再是必需品，而只是一个选项。

或许在不久的将来，程序员的工位上不再是三块显示器，而只剩下一杯咖啡和一部手机。
